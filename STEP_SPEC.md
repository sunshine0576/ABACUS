# 算盘演示与交互规范（UI层）

## 1. 文档定位（避免冲突）

- 本文档只定义 **UI层行为**：初始化呈现、播放控制、交互边界、手动拨珠、提示文案。
- 计算拆解、动画派生、脚本生成统一以 `CALC_ENGINE_SPEC.md` 为准。
- 发生冲突时，按以下优先级处理：
  1. 计算与脚本语义：`CALC_ENGINE_SPEC.md`
  2. 播放与交互体验：`STEP_SPEC.md`
- 手动拨珠与演示播放分离：手动无动画，演示有动画。

## 2. 算盘结构规范

- 每档使用标准珠型：上 2、下 5。
- 只有靠近横梁的珠子参与计数。
- 当前教学数字范围仍为十进制 0~9（每档教学态使用 1 颗上珠 + 0~4 颗下珠）。

## 3. 脚本消费约束（UI只执行）

- UI 层输入必须是已生成的 `RunScript`。可选两种消费视图：
  - `RunScript.frames[]`：扁平帧序，适合线性自动播放；
  - `RunScript.scenes[]`：分镜结构（场=一个运算位），适合按位回放、跳场、场间停顿等高级 UI 行为。
- 两种视图语义一致，每帧都带 `sceneId` 与 `fromCalcStep`，UI 可任选其一。
- UI 不允许再推导"下一步如何计算/如何进位借位/如何拆 beadOps"，所有珠级动作（`upperDelta`、`lowerDelta`、`beadOps`）由脚本提供。
- UI 只做：设置数值、执行动画、渲染高亮、显示文案。

## 4. 统一执行顺序（UI消费）

- 初始化展示位序：百位 -> 十位 -> 个位。
- 运算展示位序由脚本决定，默认应为：个位 -> 十位 -> 百位。

### 4.1 初始化步骤

以非零位为单位生成初始化步骤：

1. 拨入该位数值（`animate=true`，高亮变化珠）
2. 确认该位最终数字（`animate=false`，高亮该位表示珠）

示例：`12`

- 初始化十位 1
- 确认十位 1
- 初始化个位 2
- 确认个位 2

### 4.2 运算展示步骤（分镜模式）

脚本以"场"（`scene`）为单位组织运算展示，每个非零运算位对应一个场，场内典型结构：

1. **起始确认**（`focus`，`animate=false`）：高亮当前位数字，并展示该场的珠算手法标签（`technique`）
2. **拨珠动画**（`move`，`animate=true`）：可有 1~N 段，每段对应"差异档"的拨珠动作
3. **结果确认**（`confirm`，`animate=false`）：高亮当前位结果数字

示例：`27 + 18`（2 场，共 3 段动画）

- 场 1（个位场，`进位加`）：focus(7) → move(个位下-2) → move(十位下+1) → confirm(5)
- 场 2（十位场，`直加`）：focus(3) → move(十位下+1) → confirm(4)

执行顺序公理（与计算规范一致，UI 不得违反）：

- **先右后左**：场内 `move` 按 `placeIndex` 升序，即从个位向高位推进；
- **先上后下**：单段 `move` 内 `beadOps` 总是先动上珠组、再动下珠组。

### 4.3 珠算中文术语（UI文案约束）

通用珠态用语：

- `上`：上珠靠梁（值 5）。
- `下`：下珠靠梁（值 1）。
- `进`：向高位 +1。
- `退`：本位回拨归并（手动归一化时使用）。
- `借`：向高位 -1。

六大珠算手法（与 `focus.technique` 对齐，文案优先使用这些标签）：

- `直加`：仅下珠 +N。
- `凑五加`：上珠 +1 同时下珠 -m（如 4+3）。
- `进位加`：跨档加法，含向高位 +1（如 7+5）。
- `直减`：仅下珠 -N。
- `破五减`：上珠 -1 同时下珠 +m（如 7-3）。
- `退位减`：跨档减法，含向高位 -1（如 12-5）。

UI 文案优先使用上述术语；禁止仅用"移动/变化"替代珠算动作词。

## 5. 交互边界

- 下一步在最后一步：提示“已到最后一步”。
- 上一步在第一步：提示“已到第一步”。
- 若当前正在动画，点击“下一步”时应直接终止当前动画并立即到位，再进入下一步骤。

## 6. 播放策略

- 自动播放每步延迟必须大于单步动画时长。
- 自动播放按 **全局帧序** 推进；相邻两帧之间的等待由脚本在 `RunScript.frames` 上区分：**场切换**（`sceneId` 变化）与 **进入动画帧**（`type === "move"`）使用不同延迟常量，由主程序实现（见 `app.js` 中 `AUTO_PLAY_DELAY_*`）。
- 演示模式下「下一步」按 **场** 推进：一次点击播完当前 `scene` 内全部帧（`focus` → 若干 `move` → `confirm`），帧间节奏使用 `MANUAL_SCENE_DELAY_*`。
- 自动播放推进时若检测到动画未结束，应短轮询等待，避免并发推进。
- 每步高亮在固定时长后自动清除，避免界面残留。

## 7. 手动拨珠计数与进位归一化规则

- 手动拨珠后，先按当前珠态计算“原始值”，再立即执行归一化重排。
- 归一化目标：同一位最终保持教学标准表达（0~9），超出部分向高位进位。
- 典型规则：
  - 下珠达到 5 颗：自动归并为当前位的 `5`（表现为上珠 1、下珠 0）。
  - 上珠达到 2 颗：自动向高位进 `1`，当前位清零。
- 归一化必须是无动画即时生效，避免手动操作时视觉延迟。
- 归一化只影响数值表达，不改变“手动模式无动画、演示模式有动画”的总原则。

## 8. 手动计数验收样例

- 个位从 4 再拨 1 个下珠：应从 `4` 变为 `5`（上珠1、下珠0）。
- 个位从 9 再拨 1 个上珠（凑到上珠2）：应进位为十位 +1，个位归 0。
- 十位发生相同归并/进位时，规则完全一致（按位独立、向更高位进位）。

## 9. 最高位边界与溢出处理

- 当前五档教学范围为 `0~99999`，`99999` 为最大可表示数（万位 / 千位 / 百位 / 十位 / 个位）。
- 若手动拨珠导致原始值超过 `99999`（例如万位出现两个 5 触发继续进位）：
  - 本次操作应被回退到拨珠前状态；
  - 界面提示「已到最大可表示数，本次拨珠未生效」；
  - 不允许进入错误中间态或显示异常珠型。
